# Privacy Policy

Last updated: 2026-03-26

This Privacy Policy applies to the "Meta Ads Monitoring Bot" repository and related tooling.

## 1) What This App Does

This app reads advertising and page/post metadata from Meta APIs to support ad monitoring, optimization decisions, and test-post detection for configured shops.

## 2) Data We Collect and Process

Depending on enabled features and granted scopes, the app may process:

- Ad account data (for example ad IDs, ad set IDs, campaign IDs, spend, clicks, CPC, budget).
- Creative metadata (for example `effective_object_story_id`, created time).
- Page/actor content metadata (for example post/media ID, created time, text snippet/message when available).
- Configuration and operational metadata (shop mapping, strategy tags, optimization outputs).

The app is not designed to intentionally collect sensitive personal data categories.

## 3) Data Sources

- Meta Graph API / Marketing API.
- Operator-managed local files (for example `config.json`).
- Operator-managed sheets/files (for example Google Sheets outputs and local JSON artifacts).

## 4) Why We Process Data

- Monitor ad performance and pacing.
- Generate optimization suggestions and machine-readable operation commands.
- Detect newly published, not-yet-promoted posts for test workflows.
- Provide diagnostics and operational logs for troubleshooting API access and permission issues.

## 5) Legal Basis and Permissions

The app operates only with credentials provided by the account operator. The operator is responsible for obtaining lawful basis/authorization and for granting only required Meta scopes (for example `ads_read`, page read scopes).

## 6) Data Sharing and Third Parties

We do not sell personal data. Data is only sent to services explicitly used by the operator (for example Meta APIs and optionally Google Sheets).

## 7) Data Retention

Retention is controlled by the operator environment and platform settings. Local output files (for example `pending_tests.json`, optimization artifacts, logs) can be deleted by the operator at any time.

## 8) Security

- Secrets (tokens, keys) should be stored in environment variables and never committed to source control.
- Access should follow least privilege.
- Operators should rotate credentials periodically and revoke compromised tokens immediately.

## 9) User Rights

Where applicable, users may request access, correction, or deletion of their data through the account/page operator who controls the relevant Meta assets.

## 10) Data Deletion Instructions (Meta Requirement)

If you want data generated by this app removed:

1. Revoke the app token/permissions in Meta Business/App settings.
2. Delete local files and outputs created by this app (including logs and JSON artifacts).
3. Remove rows/records written by this app from connected data stores (for example Google Sheets) if applicable.

Deletion requests can be submitted via the repository issue tracker with subject: "Data Deletion Request".  
Repository: `https://github.com/ngaiwachoi-debug/metaadsbot`

## 11) International Transfers

Data processing/storage location depends on operator infrastructure and the cloud services used by the operator (for example Meta/Google infrastructure).

## 12) Children

This app is not intended for children under 13 and is intended for business/operational advertising use.

## 13) Changes to This Policy

We may update this policy from time to time. The "Last updated" date reflects the latest revision.

## 14) Contact

For privacy, deletion, or compliance questions, open an issue in this repository:  
`https://github.com/ngaiwachoi-debug/metaadsbot/issues`
